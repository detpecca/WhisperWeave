# Product

## Register

product

## Users

个人知识工作者。他们脑子里随时会冒出碎片想法、待办、读后感、会议要点，却没精力立刻整理成文。他们的痛点是：零碎笔记散落在备忘录、聊天框、便签里，事后要花大量时间翻找、拼接、改写成可分享的文档。使用场景是「随手记 → 积累 → 一键织成结构化文档 → 导出或同步到飞书归档/分享」。多在桌面浏览器上使用，追求快和顺手，不想被工具本身打断思路。

## Product Purpose

WhisperWeave（絮语织）把零碎、口语化的碎片输入，借助 LLM 一键织成结构化、详细、可直接阅读与分享的 Markdown 文档。核心闭环：碎片化输入 → 智能聚合 → 结构化输出 → 人工微调 → 导出 / 分享。它不是另一个笔记库，而是一条「把碎语织成文章」的流水线——降低从想法到成文的门槛，让个人知识真正沉淀成可复用的文档。成功标准：用户从记下碎片到拿到一篇成稿，中间几乎不需要手动组织。

## Brand Personality

高效、利落、克制。像一把顺手的好工具，不喧宾夺主：界面退到背景里，让用户的文字成为主角。语气直接、不啰嗦，按钮和标签说人话。视觉上追求工具感与秩序感，而非装饰性。参考 Linear 的利落、Notion 的安静，但要更轻、更专注单一任务。

## Anti-references

- 拥挤的仪表盘式界面：一屏塞满图表、侧栏、通知、徽章，信息密度过高，让人焦虑。WhisperWeave 是单一任务工具，不是 cockpit。
- AI slop 视觉：紫色/青色霓虹渐变、发光描边、厚单边色块、渐变标题文字——这些是 AI 生成 UI 的典型 tell，看起来花哨实则廉价。
- 过度装饰的「创意工具」外壳：拟物纸张、手写字体、繁复插画，喧宾夺主，盖过内容本身。

## Design Principles

1. **内容是主角，界面是舞台**：任何装饰一旦抢了用户文字的注意力就是失败。卡片、阴影、配色都要退到背景里。
2. **单一任务流，不要 cockpit**：一次只引导用户做一件事——记碎片，或织文档。拒绝把所有功能摊在一屏。
3. **快过思考**：从输入到成稿的路径要短到几乎不存在。默认值要敢做决定，减少需要用户配置的项。
4. **克制的秩序感**：用留白和层级组织信息，而非边框和分隔线。圆角、阴影、色阶都要节制，不堆砌。
5. **说人话**：按钮、标签、空状态文案直接、具体、不卖弄。宁可两个字说完，不写一句话。

## Accessibility & Inclusion

当前阶段以视觉打磨为主，暂不强制 WCAG 达标。但仍遵循基础底线：正文文字对比度足够阅读、交互元素键盘可达、不依赖纯颜色传达状态。后续如转向正式产品再补全无障碍审计。
